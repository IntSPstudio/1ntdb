## Do you 1nt database?

This is my idea / vision of a database system that I have been planning and developing for a long time. It is based on a relational database with its own twist. The idea came from hobbies many years ago when I was looking for a ready-made database of this kind to replace the currently used Access database. The most important feature would be the ability to use the software in a browser on many different devices and to import data, for example, through an API interface.

## Roadmap

This is such a large project, and I am intentionally creating a separate side track for experimentation and development.

This folder, called 'sideproject', contains a minimized and customized version of the system. The purpose is to help guide development in the right direction while I iterate on the core architecture.

I have been planning this project for a long time, and I am now focusing on building a Python-based backend. The development will start with a CLI interface, then move to API endpoints, and eventually expand into a web-based graphical interface.
